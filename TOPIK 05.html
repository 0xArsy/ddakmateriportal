Topik 05 Combinational MSI Sequential Logic

Combinational Circuits: MSI
& Sequential Logic

Pelajari empat komponen MSI utama — Decoder, Encoder, Multiplexer, Demultiplexer — serta fondasi Sequential Logic: Latch dan Flip-Flop dalam satu halaman interaktif.

Decoder
n input → 2ⁿ output
Encoder
2ⁿ input → n output
Multiplexer
2ⁿ input → 1 output
Demultiplexer
1 input → 2ⁿ output

MSI (Medium Scale Integrated) Circuits

KEY TERM: MSI Circuit

Medium Scale Integrated Circuit adalah rangkaian logika kombinasional yang sudah terpadu. Empat komponen MSI yang paling umum digunakan adalah Decoder, Encoder, Multiplexer, dan Demultiplexer. Masing-masing berfungsi melakukan konversi antara kode dan entitas, atau antara satu jalur dan banyak jalur.

MENTAL MODEL: Analogi Sehari-hari
Decoder = Kamus → kode masuk, arti keluar
Encoder = Keyboard → huruf masuk, kode biner keluar
MUX = Remote TV → pilih saluran mana yang ditampilkan
DEMUX = Router → satu kabel, distribusikan ke banyak perangkat

Perbandingan 4 Komponen MSI

Komponen Input Line Selector Output Fungsi Utama
Decoder n bit — maks 2ⁿ Kode → Entitas
Encoder maks 2ⁿ — n bit Entitas → Kode
Multiplexer 2ⁿ data n bit 1 Banyak → Satu (Data Selector)
Demultiplexer 1 n bit 2ⁿ Satu → Banyak

Decoder

KEY TERM: Decoder

Mengkonversi informasi biner dari n input menjadi maksimal 2ⁿ output. Setiap output merupakan minterm dari variabel-variabel input. Disebut juga n-to-m-line decoder atau n×m decoder dengan m ≤ 2ⁿ.

// Setiap output = minterm 2-variabel

F₂ = X·Y'    // input 10
F₃ = X·Y     // input 11

💡 Hanya satu output yang bernilai HIGH (1) pada setiap waktu, sesuai dengan kode input yang diberikan.

COMMON PITFALL

Pada 0-enable (active-low), decoder aktif saat E'=0, BUKAN saat E'=1. Kebanyakan MSI decoder di dunia nyata menggunakan sinyal active-low untuk enable.

Membangun Decoder Lebih Besar dari yang Kecil

3×8 dari dua 2×4 (1-enable) + Inverter

Input MSB (w) → inverted ke enable decoder pertama; langsung ke enable decoder kedua. x,y → masuk ke S₁,S₀ kedua decoder.

4×16 dari dua 3×8 (1-enable)

Input MSB (w) → enable decoder pertama; inverted → enable decoder kedua. x,y,z → S₂,S₁,S₀ kedua decoder.

// Prinsip: 1 Decoder (n×2ⁿ) → 2 Decoder ((n-1)×2^(n-1))
// Contoh dari soal latihan:
1×2 Decoder → 2× 1×2 → 4× 1×2 → 8× 1×2
2×4 Decoder → 4× 2×4 // untuk membuat 4×16
1×2 Decoder → 2× 1×2 → 4× 2×4 // cara alternatif
Prinsip Implementasi Fungsi via Decoder

Karena setiap output decoder = minterm, kita bisa mengimplementasikan sembarang fungsi kombinasional n-input menggunakan decoder n:2ⁿ + OR gate. Tinggal hubungkan output yang merupakan minterm fungsi ke OR gate.

Contoh: Full Adder dengan 3×8 Decoder

Fungsi:

S(x,y,z) = Σm(1,2,4,7)
C(x,y,z) = Σm(3,5,6,7)

Implementasi:

  • → Output decoder 1,2,4,7 → OR gate → S
  • → Output decoder 3,5,6,7 → OR gate → C

Encoder

KEY TERM: Encoder

Kebalikan dari decoder. Diberikan sekumpulan input line (hanya satu yang aktif HIGH), encoder menghasilkan kode biner yang bersesuaian. Berisi 2ⁿ atau kurang input line dan n output line. Diimplementasikan dengan OR gate.

COMMON PITFALL: Encoder Biasa vs Priority Encoder

Encoder biasa hanya valid jika tepat satu input = 1. Jika ada lebih dari satu input aktif bersamaan, outputnya tidak valid. Untuk kondisi ini, digunakan Priority Encoder yang memberikan prioritas pada input dengan nomor tertinggi.

Contoh: 8-to-3 (Octal-to-Binary) Encoder

Multiplexer (MUX) & Demultiplexer (DEMUX)

KEY TERM: Multiplexer

Disebut juga data selector. Memiliki 2ⁿ jalur input, n jalur selection, dan 1 jalur output. Berdasarkan nilai n-bit selector, salah satu input diteruskan ke output.

4-to-1 MUX – Truth Table

S₁S₀Y
00I₀
01I₁
10I₂
11I₃

Persamaan Output MUX

// Output = sum of (data × selection)
Y = I₀·(S₁'·S₀')
  + I₁·(S₁'·S₀)
  + I₂·(S₁·S₀')
  + I₃·(S₁·S₀)

// MUX = Decoder + AND gates + OR gate
// n:2ⁿ decoder → tambah 2ⁿ AND gate

8-to-1 MUX dari MUX Lebih Kecil

Cara 1: Dua 4:1 MUX + Satu 2:1 MUX

// S₂ → select untuk 2:1 MUX (luar)
// S₁, S₀ → select untuk kedua 4:1 MUX
MUX-A (I₀..I₃) → F₀..₃ [pilih via S₁,S₀]
MUX-B (I₄..I₇) → F₄..₇ [pilih via S₁,S₀]
MUX-C (F₀..₃, F₄..₇) → Y [pilih via S₂]

Cara 2: Empat 2:1 MUX + Satu 4:1 MUX

// S₀ → select untuk 4 MUX 2:1 (dalam)
// S₂,S₁ → select untuk 4:1 MUX (luar)

16-to-1 MUX: Lima 4:1 MUX

// 4 MUX 4:1 (dalam, pilih via S₁,S₀)
// 1 MUX 4:1 (luar, pilih via S₃,S₂)
TIPS

Dari soal latihan: Untuk membangun 8:1 MUX bisa menggunakan 4 MUX 2:1 → 2 MUX 2:1 → 1 MUX 2:1, atau 2 MUX 4:1 → 1 MUX 2:1.

Implementasi Fungsi Boolean via MUX

Ekspresikan fungsi dalam bentuk sum-of-minterms

Hubungkan n variabel ke n selection line MUX 2ⁿ:1

Pasang '1' pada data line yang merupakan minterm fungsi, '0' untuk yang bukan

Contoh: F(A,B,C) = Σm(1,3,5,6)

// Gunakan 8:1 MUX, connect A→S₂, B→S₁, C→S₀
I₀ = 0  // minterm 0 bukan bagian fungsi
I₁ = 1  // minterm 1 ✓ (m1)
I₂ = 0  // minterm 2 tidak ada
I₃ = 1  // minterm 3 ✓ (m3)
I₄ = 0  // minterm 4 tidak ada
I₅ = 1  // minterm 5 ✓ (m5)
I₆ = 1  // minterm 6 ✓ (m6)
I₇ = 0  // minterm 7 tidak ada

// Output Y = m₁ + m₃ + m₅ + m₆ = F(A,B,C) ✓

Kebalikan MUX. Satu data input diarahkan ke salah satu dari 2ⁿ output berdasarkan nilai selection line. Rangkaian DEMUX identik dengan Decoder + Enable!

Sequential Logic – Pengantar

Combinational Circuit

Output hanya bergantung pada input saat ini (present input). Tidak ada memori.

output = f(input)
Sequential Circuit

Output bergantung pada present input DAN state sebelumnya. Memiliki elemen memori.

output = f(input, state)
state(baru) = g(input, state)

Tipe-tipe Sequential Circuit

Synchronous

Output berubah hanya pada waktu tertentu (dipicu clock)

Asynchronous

Output berubah kapan saja ketika input berubah

Bistable (Multivibrator)

2 kondisi stabil → dasar dari Latch & Flip-Flop

Memory Element – Karakteristik Dasar

Perintah (di waktu t)Q(t)Q(t+1)
SetX1
ResetX0
No Change00
No Change11

Q(t) = current state (state saat ini)
Q(t+1) atau Q⁺ = next state (state berikutnya)

Tipe triggering:
• Pulse-triggered → Latch (level-sensitive)
• Edge-triggered → Flip-Flop (positive/negative edge)

Latch

KEY TERM: Latch vs Flip-Flop
Latch (Asynchronous)

Output langsung berubah saat input berubah. Tidak menunggu clock. Level-sensitive (aktif selama EN = HIGH atau LOW).

Flip-Flop (Synchronous)

Output berubah hanya pada edge clock (rising atau falling). Edge-triggered.

Active-High S-R Latch (NOR gate)

SRQQ'Keterangan
00NCNCNo Change
1010SET
0101RESET
1100INVALID!
COMMON PITFALL

S=1 dan R=1 secara bersamaan menghasilkan kondisi INVALID (Q=Q'=0, melanggar komplemen). Kondisi ini harus dihindari!

Active-Low S-R Latch (NAND gate)

S'R'QQ'Keterangan
0110SET
1001RESET
11NCNCNo Change
0011INVALID!

Pada active-low: latch aktif saat input = 0 (LOW). Logikanya terbalik dari active-high.

Jawaban soal #13 (active-low): Time 0: S=0,R=1 → Q=1,Q'=0 (SET). Time 1: S=1,R=1 → Q=1,Q'=0 (NC). Time 2: S=1,R=0 → Q=0,Q'=1 (RESET).

Gated S-R Latch (dengan Enable)

S-R Latch + EN input + 2 NAND gate. Output berubah hanya saat EN = HIGH.

KEY TERM: Gated D Latch

Dibuat dengan menghubungkan R = S' pada Gated S-R Latch. Mengeliminasi kondisi invalid! Saat EN=HIGH, Q mengikuti (follows) input D.

Characteristic Table – D Latch

Saat EN=1: Q(t+1) = D

// Asumsi EN selalu 1 (latch terbuka)
Time 0: D=0 → Q=0, Q'=1
Time 1: D=1 → Q=1, Q'=0
Time 2: D=0 → Q=0, Q'=1

Flip-Flop

KEY TERM: Flip-Flop

Perangkat bistable synchronous. Output berubah hanya pada edge (tepi) sinyal clock: positive edge (LOW→HIGH) atau negative edge (HIGH→LOW). Berbeda dengan latch yang level-sensitive.

Characteristic Table – S-R Flip-Flop

↑ = clock transition LOW → HIGH (positive edge)

SRCLKQ(t+1)Keterangan
00XQ(t)No Change
01↑0Reset
10↑1Set
11↑?Invalid!

// Asumsi Q awal = ?
Time 0: S=0,R=1 → Q=0, Q'=1 (Reset)
Time 1: S=0,R=0 → Q=0, Q'=1 (No Change)
Time 2: S=1,R=0 → Q=1, Q'=0 (Set)

Keunggulan J-K FF

Tidak ada kondisi invalid! Saat J=K=1 → Toggle (output berbalik). Ini adalah perbedaan utama dari S-R FF.

Time 0: J=0,K=1 → Q=0, Q'=1 (Reset)
Time 1: J=1,K=1 → Q=1, Q'=0 (Toggle dari 0)
Time 2: J=0,K=0 → Q=1, Q'=0 (No Change)
Time 3: J=1,K=0 → Q=1, Q'=0 (Set=tetap)

// Persamaan karakteristik T FF:
Q(t+1) = T⊕Q  (XOR)
// Jawaban soal #18 (T FF, Q awal=1):

PRESET (PRE) dan CLEAR (CLR) atau direct set (SD) dan direct reset (RD).

Soal 1 Decoder

Sebuah 4×16 decoder ingin dibangun dari decoder 2×4 (dengan enable). Berapa jumlah minimum decoder 2×4 yang dibutuhkan?

A2 decoder
B4 decoder
C5 decoder
D8 decoder
Benar!
Salah. Coba lagi!

Metodologi Penyelesaian:

4×16 decoder memiliki 4 input dan 16 output. Untuk membangun dari 2×4 decoder:

2 bit MSB (bit 3,2) digunakan untuk memilih kelompok decoder mana yang aktif. Ini membutuhkan 1 decoder 2×4 sebagai selector, yang menghasilkan 4 sinyal enable.

2 bit LSB (bit 1,0) masuk ke masing-masing dari 4 decoder 2×4 sebagai address. Total output: 4 × 4 = 16. ✓

Total: 1 (selector) + 4 (decoder data) = 5 decoder 2×4. Jawaban: C.

2-bit MSB → 1× Decoder 2×4 (menghasilkan 4 enable)
2-bit LSB → 4× Decoder 2×4 (masing-masing dengan 1 enable)
Total = 1 + 4 = 5 decoder 2×4

AI₃ = 0
BI₃ = 1
CI₃ = A
DI₃ = C
Benar!
Salah. Coba lagi!

Metodologi Penyelesaian:

F(A,B,C) = Σm(1,2,4,7) berarti fungsi = 1 hanya pada minterm 1, 2, 4, dan 7.

Pada 8:1 MUX, input data I_k bernilai 1 jika minterm ke-k ada dalam fungsi, dan 0 jika tidak.

I₄=1 I₅=0 I₆=0 I₇=1
// m3 tidak ada di {1,2,4,7} → I₃ = 0
Soal 3 J-K Flip-Flop

Sebuah J-K Flip-Flop memiliki Q saat ini = 1. Jika J=1 dan K=1 diberikan pada positive edge clock berikutnya, nilai Q(t+1) adalah?

AQ = 1 (tetap)
BQ = 0 (Toggle)
CQ = Invalid
DQ = 0 (Reset)
Benar!
Salah. Coba lagi!

Metodologi Penyelesaian:

J=1, K=1 → kondisi Toggle pada J-K Flip-Flop.

Toggle artinya: Q(t+1) = Q(t)' = komplemen dari Q saat ini.

Q(t) = 1 → Q(t+1) = 1' = 0.

J=K=1 → Toggle
Q(t)=1 → Q(t+1) = 0
// Ini keunggulan JK FF: tidak ada kondisi invalid!
Soal 4 S-R Latch

Pada active-high S-R Latch dengan Q saat ini = 0 dan Q' = 1, apa yang terjadi jika S=0 dan R=0 diberikan?

AQ berubah menjadi 1 (SET)
BQ tetap 0, Q' tetap 1 (No Change)
CQ=0, Q'=0 (Invalid)
DQ berubah menjadi 0 (RESET)
Benar!
Salah. Coba lagi!

Metodologi Penyelesaian:

Pada characteristic table S-R Latch: S=0, R=0 → kondisi No Change (Memorise).

Latch akan mempertahankan state sebelumnya: Q=0, Q'=1.

Inilah fungsi "memory" dari latch — mampu menyimpan informasi saat tidak ada perintah baru.

Apa perbedaan Latch dan Flip-Flop?

Latch: Asynchronous, output langsung berubah saat input berubah (level-sensitive).

Flip-Flop: Synchronous, output berubah hanya pada clock edge (edge-triggered).

Apa kondisi Toggle pada J-K FF?

J=1, K=1 → Toggle: Q(t+1) = Q(t)'

Ini membuat J-K FF lebih unggul dari S-R FF karena tidak ada kondisi invalid.

Mengapa DEMUX sama dengan Decoder + Enable?

Pada DEMUX: Data D masuk ke Enable, selection line ke address decoder. Rangkaian dan persamaan Boolean-nya identik.

Y₀ = D·S₁'·S₀' = E·X'·Y'

Rumus Q(t+1) untuk T Flip-Flop?

Q(t+1) = T ⊕ Q(t)

T=0 → No Change (Q tetap)
T=1 → Toggle (Q berbalik)

Bagaimana mengimplementasikan fungsi Boolean dengan MUX?

1. Ekspresikan sebagai sum-of-minterms
2. Hubungkan n variabel ke n selection line
3. I_k = 1 jika minterm k ada, 0 jika tidak

Output = fungsi yang diinginkan ✓

Apa yang dimaksud Asynchronous Input PRE dan CLR?

Input yang mempengaruhi state FF tanpa menunggu clock.

PRE → Force Q=1 (Preset)
CLR → Force Q=0 (Clear)
Normal mode: PRE=CLR=0 (active-low)
