1. Gambaran Besar: Von Neumann Architecture

💡 Mental Model

Bayangkan komputer sebagai sebuah pabrik: ada gudang bahan baku (Memory), mesin produksi (Datapath/ALU), manajer (Control Unit), dan conveyor belt (Bus) yang mengangkut data antar bagian.

Memory

Menyimpan program dan data. Diakses oleh Datapath melalui bus.

Datapath

Komponen yang melakukan operasi pada data — berisi register, ALU, dan shifter.

Control Unit

Mengirim sinyal kendali yang menentukan operasi datapath, memory, dan I/O.

🔑 Key Terms

ISA (Instruction Set Architecture)

Spesifikasi komputer dari sudut pandang programmer di level terendah. Berfungsi sebagai interface antara hardware dan software.

Computer Architecture

Deskripsi high-level dari hardware yang mengimplementasikan komputer berdasarkan ISA. Mencakup kecepatan, biaya, dan keandalan.

Microoperation

Operasi dasar yang dilakukan pada data yang tersimpan di register. Contoh: R0 ← R1 + R2

Register File

Sekumpulan register dengan shared access resources (multiplexer, decoder). Tampak seperti memory berbasis flip-flop.

Control Word

Kumpulan sinyal kontrol yang terorganisir. Untuk mengeksekusi microinstruction, kita mengaplikasikan nilai control word selama satu clock cycle.

Bus

Jalur komunikasi (shared transfer path) antar komponen. Terdiri dari data bus, control bus, address bus. Lebar bus menentukan kapasitas transfer.

⚠️ Common Pitfalls

  • • ISA ≠ Computer Architecture: ISA adalah "tampilan" komputer bagi programmer; arsitektur adalah implementasi hardware-nya. Keduanya saling terkait tapi berbeda level abstraksi.
  • • Data bus ≠ Address bus: Lebar data bus = ukuran word (data yang diproses). Lebar address bus menentukan berapa banyak lokasi memori yang bisa diakses (n-bit → 2ⁿ alamat).
  • • Datapath ≠ ALU: Datapath adalah sistem lengkap (register file + ALU + shifter + bus + mux). ALU hanyalah satu komponen di dalamnya.
  • • Clock period vs. Clock frequency: Keduanya kebalikan satu sama lain. Frekuensi tinggi = period pendek = lebih banyak siklus per detik.

2. Datapath: Komponen & Cara Kerja

Sebuah datapath dispesifikasikan oleh tiga hal: (1) sekumpulan register, (2) microoperations yang dilakukan pada data di register, dan (3) control interface.

Komponen Register

  • • Individual registers — register terpisah (R0, R1, R2, R3)
  • • Register file — sekumpulan register dengan akses bersama via mux/decoder
  • • Kombinasi keduanya

Shared Resources

  • • Bus — jalur transfer data bersama (Bus A, Bus B, Bus D)
  • • ALU — shared resource untuk aritmatika & logika
  • • Shifter — shared resource untuk operasi shift
  • • Multiplexer (MUX) — pemilih jalur data

Contoh: Microoperation R0 ← R1 + R2

1 Aplikasikan 01 ke A select → konten R1 ke Bus A
2 Aplikasikan 10 ke B select + 0 ke MB select → konten R2 ke Bus B
3 Aplikasikan 0010 ke G select → ALU melakukan penjumlahan: G = Bus A + Bus B
4 Aplikasikan 0 ke MF select + 0 ke MD select → nilai G ke Bus D
5 Aplikasikan 00 ke Destination select → aktifkan Load input ke R0
6 Aplikasikan 1 ke Load Enable → R0 di-load pada clock pulse
⏱ Total: 1 clock cycle untuk menyelesaikan satu microoperation.

3. Arithmetic Logic Unit (ALU)

Arithmetic Circuit

n-bit parallel adder + B input logic. Menangani operasi: transfer, increment, add, subtract, decrement.

Logic Circuit

Implementasikan: AND, OR, XOR, NOT per bit menggunakan custom circuit (lebih efisien dari MUX).

Selector (S₂)

S₂=0 → pilih arithmetic circuit. S₂=1 → pilih logic circuit. Diseleksi via 2-to-1 MUX.

⚠️ Tentang B Input Logic

Hanya ada 4 fungsi B yang bisa dipilih sebagai Y dalam G = A + Y: all 0s, B, B̄ (komplemen B), all 1s. Kombinasi dengan Cin=0 atau 1 menghasilkan 8 fungsi aritmatika berbeda.

Logic Circuit — Custom Design (lebih efisien)

Formula: Gᵢ = S̄₀AᵢBᵢ + S̄₁AᵢB̄ᵢ + S₀AᵢB̄ᵢ + S₁S̄₀Āᵢ

4. Shifter

Basic Left/Right Shifter

(S₁,S₀) = 00Pass B unchanged
(S₁,S₀) = 01Right shift
(S₁,S₀) = 10Left shift
(S₁,S₀) = 11Unused
Serial Input: I_R (right), I_L (left)
Serial Output: R = MSB input, L = LSB input

Barrel Shifter

Rotate: bit yang keluar dimasukkan ke posisi kosong.

S = 00Posisi tidak berubah
S = 01Rotate left 1 posisi
S = 10Rotate left 2 posisi
S = 11Rotate left 3 posisi
Large barrel shifter → layers of MUX (contoh 64-bit: layer 1 shift 0/16/32/48, layer 2 shift 0/4/8/12, layer 3 shift 0/1/2/3)

5. ISA, CPU & Moore's Law

CPU = Control + ALU + Registers

  • Control Unit — monitor & arahkan urutan instruksi
  • ALU — operasi aritmatika & logika (add, sub, and, or, not, inc...)
  • Registers — memori cepat dalam CPU (PC, ACC, IR, MAR, MDR)

Siklus eksekusi instruksi:

Fetch → Decode → Execute → Store

Moore's Law

Gordon Moore (Intel co-founder) memprediksi tahun 1965:

Densitas transistor akan berlipat ganda setiap 18 bulan → peningkatan clock frequency.

Dari 1971–2005: total 36.000× peningkatan. Analoginya: perjalanan LA ke NYC bisa dalam hitungan detik dengan hanya beberapa sen!

Roadblock: Power density jadi masalah — buruk untuk pendinginan, keandalan, dan lingkungan.

Q1 Konseptual

Sebuah komputer memiliki address bus 32-bit. Berapa banyak lokasi memori yang dapat dialamatkan?

📐 Metodologi

Lebar address bus menentukan jumlah alamat yang dapat diakses. Formula: jumlah alamat = 2^n di mana n = lebar bus.

Untuk n=32: 2³² = 4.294.967.296 ≈ 4 GB lokasi memori. Itulah mengapa sistem 32-bit hanya bisa mengakses maksimal 4 GB RAM secara langsung.

Contoh lain: n=16 → 2¹⁶ = 65.536 alamat; n=64 → 2⁶⁴ = jumlah yang sangat besar (16 exabyte).

Q2 ALU / Control Word

Untuk melakukan microoperation R1 ← R2 − R3, nilai FS (Function Select) yang benar adalah?

📐 Metodologi: Two's Complement Subtraction

Pengurangan dalam hardware dilakukan dengan two's complement: A − B = A + B̄ + 1

• B̄ = komplemen (NOT) dari B → dicapai dengan S₁=1, S₀=0 pada B input logic → Y = B̄

• +1 dari Cin=1 → dalam FS, Cin dikodekan sebagai bit S₀ yang terlibat

Dari Table 9-4/9-5: FS = 0101 → G Select = 0101 → operasi G = A + B̄ + 1 = A − B ✓

Full control word: DA=001 (R1), AA=010 (R2), BA=011 (R3), MB=0, FS=0101, MD=0, RW=1

Q3 Shifter

Pada 4-bit basic left/right shifter, jika input B = 1011 dan (S₁,S₀) = 01 dengan I_R = 0, maka output H adalah?

📐 Metodologi: Right Shift

(S₁,S₀) = 01 → Right shift operation.

Pada right shift: setiap bit bergeser ke kanan 1 posisi.

B = 1 0 1 1

B3 B2 B1 B0

Setelah right shift: bit MSB diisi oleh I_R (serial input untuk right shift)

H = I_R B3 B2 B1 = 0 1 0 1 = 0101 ✓

B0 (bit paling kanan) keluar sebagai serial output R = B0 = 1

Q4 Control Word

Microoperation R4 ← Data in memerlukan nilai MD dan RW berturut-turut:

📐 Metodologi: Load dari External Data

MUX D memilih antara dua sumber untuk Bus D:

• MD=0 → Bus D menerima output dari Function Unit (hasil ALU/Shifter)

• MD=1 → Bus D menerima Data in (dari eksternal/memori) ← ini yang kita butuhkan

Setelah data ada di Bus D, kita ingin menulis ke register R4:

• RW=1 → aktifkan Register Write

• DA=100 (R4) → R4 adalah register tujuan

Ini adalah operasi "memory read → register load": MD=1 mengalihkan data dari input eksternal ke Bus D, kemudian RW=1 menyimpannya ke register.

Q5 Analisis

Manakah pernyataan yang BENAR mengenai perbedaan ISA dan Computer Architecture?

📐 Metodologi: Memahami Layer Abstraksi

ISA (Instruction Set Architecture): Seperti "kontrak" antara software dan hardware. Mendefinisikan opcodes, register yang terlihat programmer, mode addressing — pokoknya apa yang bisa dilakukan komputer dari sudut pandang assembly programmer.

Computer Architecture: Cara hardware mengimplementasikan kontrak ISA tersebut. Termasuk kecepatan, biaya, keandalan, dan pertimbangan desain lainnya.

Analogi: ISA = spesifikasi mesin kopi (kopi bisa dibuat dengan tekanan sekian, suhu sekian). Computer Architecture = bagaimana pabrik merancang mesin tersebut agar memenuhi spesifikasi dengan efisien.

Dua komputer bisa punya ISA yang sama (misal x86) tapi Computer Architecture yang berbeda (Intel vs AMD). Program yang sama bisa berjalan di keduanya!
